Provisioning and using your custom RHDH configuration

This guide details the steps for provisioning and utilizing a custom configuration for your Red Hat Developer Hub instance. Following these instructions allows you to tailor the deployment to your specific needs. This includes setting up external services and optimizing performance.

Setting up RHDH with your custom configuration

Begin by defining the custom parameters for your RHDH instance in your configuration file. This file dictates how the hub will be provisioned, including resource allocation and plugin settings. Ensure that your configuration is validated against the schema to prevent deployment errors.

Configuring external PostgreSQL databases

For a production environment, it is highly recommended to use an external PostgreSQL database. You can configure this using either the Operator or the Helm Chart, depending on your deployment method. Migrating from a local database involves exporting your data and importing it into the new external instance.

Configuring RHDH deployment when using the Operator

When deploying with the Operator, custom configurations are managed through the `Backstage` Custom Resource (CR). You can specify database connections, backend parameters, and plugin configurations directly within the CR. Apply the changes to your cluster for the Operator to reconcile the state.
